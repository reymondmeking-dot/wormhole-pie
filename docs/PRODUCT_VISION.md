# 虫洞派 / Wormhole Pie：产品愿景

## 一句话定义

虫洞派（Wormhole Pie）是一组常驻 Windows 与 macOS 桌面的本地优先 AI 宠物。用户只需说出目标，宠物就能整理桌面、检索文件、记录待办与意见、打开应用和网页，并把适合的工作交给 Codex、Claude Code、Hermes、Cursor、OpenClaw 或微信适配器完成。

虫洞派不是把终端、任务编排器或模型控制台搬到桌面，而是把复杂能力藏进“虫洞”，只把清楚、安心、可操作的结果带回来。

## 品牌与产品气质

- 中文品牌统一使用「虫洞派」，英文品牌统一使用「Wormhole Pie」。
- “虫洞”代表跨文件、应用、浏览器、设备与 Agent 的连接；“派”既是可爱的宠物小队，也是分工协作的执行团队。
- 产品表达自然、温暖、克制，不用工程术语要求用户理解模型、工具链或工作流。
- 宠物可以活泼，但不能用卖惨、焦虑或情绪依赖驱动留存。
- 外观、声音、称呼、互动强度和通知频率均可调整或关闭。

## 产品承诺

- **结果优先**：用户描述目标，虫洞派交付结果，不要求用户管理执行过程。
- **本地优先**：桌面索引、待办、意见、权限、会话摘要和宠物成长默认保存在本机。
- **跨平台一致**：Windows 与 macOS 使用相同的产品语言、权限原则和数据模型，平台差异由系统适配层消化。
- **可控可撤销**：浏览与标签默认使用虚拟分类；只有用户明确点击“一键整理”时才移动真实文件，并在整理后提供安全复查、排除记忆与撤销。
- **能力可替换**：Codex、Claude Code、Hermes、Cursor、OpenClaw 与微信均通过适配器接入，任何单个后端都不是产品的唯一依赖。

## 结果式极简对话：不可破坏的硬规则

虫洞派面向用户的对话区只允许出现三类内容：

1. 用户消息。
2. 确实需要用户决定的权限询问或最终确认。
3. 已完成、未完成或需要用户补充信息的最终结果。

界面不得展示：

- 任务树、子任务列表或 Agent 编排图。
- 工具名称、工具调用、命令行和原始日志。
- 执行步骤、逐步计划、过程播报和百分比进度。
- 模型选择过程、内部草稿、思考链或隐藏推理。
- “正在分析第 2 步”“已调用浏览器工具”等工程化状态文本。

执行期间，宠物可以用安静的动作表达“已接住这件事”，但不在对话中持续刷进度。失败时只说明可验证的结果、必要原因和下一项可选动作，不倾倒内部错误日志。

推荐结果话术应自然、简短、可爱，例如：

- “这个活搞定啦，需要我帮主人打开看一眼嘛？”
- “桌面已经收拾清爽啦，3 个文件放进了‘待确认’。”
- “我找到两个同名报告，你想打开今天改过的那个吗？”
- “小红书草稿准备好啦。发布前还要主人亲自点一下确认。”
- “这次没打开，文件好像被移走了。要我重新找一遍吗？”

## 核心体验

### 桌面收纳与文件检索

- 实时同步桌面上的文件、文件夹和变化，并映射到清晰列表。
- 支持文档、图片、视频、压缩包、应用、文件夹及自定义分类。
- 普通浏览默认只添加虚拟分类和标签，不擅自移动真实文件；显式“一键整理”才把桌面顶层项目移入“虫洞派整理”的类型目录。
- 整理完成后给出项目级复查：保留在整理目录，或放回桌面并记住以后不再整理同名项目。
- “已忽略整理”集中展示本机排除规则。删除规则后，对应名称会重新参与下一次整理；未排除项目仍可整批撤销。
- 用户可用自然语言或本地语音说“打开昨天的项目报告”“找一下封面图”。
- 匹配唯一且安全时直接打开；存在歧义或风险时只问一个必要问题。

### 常用程序

- 桌面文件区提供独立“常用程序”分类，合并系统发现的应用入口与用户明确收藏的程序。
- 整理复查中识别到可启动项目时，可将它加入常用程序，也可选择放回桌面并加入忽略规则。
- 点击常用程序后交给系统启动服务打开；不拼接命令行参数，不把整个磁盘作为扫描范围。
- 普通文件列表中的可执行文件仍保持安全限制，不因“常用程序”功能而自动运行。

### 今日待办与桌面意见整理

- 今日待办支持创建、完成、延期、提醒以及与文件或社交发布动作关联。
- “意见箱”统一收纳灵感、反馈、临时笔记、待整理内容和文件批注。
- 用户说“记一下：首页分类太挤”即可生成意见；后续可转成待办、归档或关联文件。
- 待办、意见、通知和文件都可以通过标签与下拉筛选快速收束。

### 本地语音与指令

- 优先采用本地唤醒、VAD、ASR、关键词/拼音模糊匹配和本地 TTS。
- 桌面整理、文件打开、待办、意见、通知和社交页面跳转无需调用大模型 API。
- 开放式编程或复杂代理任务才按用户配置交给相应 Agent 适配器。

### 社交媒体与通知

- 小红书、X、抖音等入口支持打开首页、创作中心、发布页和官方可用的通知页面。
- 关键词能够区分本地文件和平台动作：“打开小红书方案”优先找文件，“打开小红书发布页”才打开网页。
- 优先采用平台官方 OAuth、开放 API 与深链接；没有稳定 API 时使用受限的浏览器连接器。
- 通知中心聚合待办到期、桌面新增文件、适配器结果和用户授权的平台通知。

## 多宠物角色

虫洞派允许用户同时拥有多只宠物。每只宠物有明确角色、外观与擅长领域，但共用一致的权限与安全规则，例如：

- **收纳型**：桌面、文件、标签、待办与意见箱。
- **工匠型**：Codex、Claude Code、Cursor 等编程与项目任务。
- **信使型**：微信、社交平台、通知与发布草稿。
- **探索型**：浏览器检索、资料归纳与跨应用查找。

用户可以指定主宠物，也可以让虫洞派在后台自动路由。无论由几只宠物协作，对话区仍只显示一次必要确认和一次最终结果，不暴露内部接力过程。

## 分支自进化

宠物成长采用“可选择的分支进化”，而不是不可控的自我改写：

- 每只宠物从基础角色出发，可沿收纳、创作、工程、沟通、陪伴等分支成长。
- 分支由真实使用偏好、任务完成质量、用户反馈和主动选择共同解锁。
- 不同宠物可以形成不同分支，用户可保留多个版本、预览、切换、回退或重置。
- 可进化内容包括外观、配色、饰品、动作、声音、结果话术和推荐能力。
- 宠物不得自行修改应用可执行代码、权限策略或安全边界。
- 新动作与外观必须来自已签名资源包或经过尺寸、透明度、性能和敏感内容校验的生成资产。

## Agent 与应用适配器

- **Codex**：适合代码修改、项目理解、验证和可交付成果生成。
- **Claude Code**：适合结构化代码任务与本地项目工作流。
- **Hermes**：适合通用工具、长时任务、记忆与跨应用操作。
- **Cursor**：通过公开 CLI、深链接或正式扩展接口打开项目、定位文件和承接编码任务。
- **OpenClaw**：通过其公开协议或本地服务承接自动化与工具任务。
- **微信适配器**：仅使用用户明确授权的官方能力、桌面深链接或受控连接器；不得读取加密数据库、窃取会话或绕过登录保护。

适配器的内部切换不向用户展示。最终结果应说明完成了什么、成果在哪里，以及是否存在需要用户处理的下一步。

## 浏览器会话与外部发布安全

- 登录状态由浏览器自己的 Profile、Cookie Store 和站点会话维护。
- 虫洞派不得复制原始 Cookie、令牌、密码、验证码或密码管理器内容到本地数据库、聊天记录或日志。
- 浏览器连接器按域名、账号、动作和有效期授权，只返回完成任务所需的裁剪后结构化结果。
- 页面结构变化、账号不明确或目标内容不一致时必须安全停止，不猜测点击。
- 发布内容、发送消息、上传文件、删除内容等对外动作必须在动作发生前进行最终确认。
- 最终确认应清楚显示平台、账号、可见范围、正文摘要和附件；确认只对该次动作有效，不得复用。

## 界面组成

- 首次启动位于当前屏幕工作区右上角的透明 iOS 风格小组件，之后可自由拖动。
- 向上收起会隐藏主窗口并留在 Windows 系统托盘或 macOS 菜单栏，不产生普通最小化窗口，也不占任务栏或 Dock。
- 可拖动、置顶/置底、点击穿透、隐藏的桌面宠物；宠物右键使用系统原生菜单，不显示 HTML 浮层。
- 今日待办、桌面文件、意见箱、通知和社交状态小组件。
- 极简对话抽屉，只呈现用户消息、必要确认和最终结果。
- 结果卡片可提供文件、链接、报告、草稿和“打开看看”等直接动作。
- 系统托盘或菜单栏入口可重新显示组件、切换宠物层级和退出应用，并配合适合 Windows 与 macOS 的原生通知。

## 平台差异与当前阶段

- Windows 是当前桌面包的主要实现与验收目标：使用系统托盘、系统工作区定位、系统应用入口和原生右键菜单。
- macOS 保持同一产品语义，但对应实现应使用菜单栏、Launch Services/`NSWorkspace`、`.app` 应用入口和目录授权；当前尚未完成安装包与完整权限流程验收。
- 浏览器预览只验证组件布局和状态，不代表原生托盘、窗口定位、程序启动或右键菜单已经通过桌面系统验收。
- 宠物处于桌面底层点击穿透模式时不会接收拖动和右键，应从托盘/菜单栏或主组件切回普通、置顶模式。

## 明确不做

- 不展示或保存模型私有思考链。
- 不把任务树、工具调用、执行步骤或进度流暴露给用户。
- 不允许适配器绕过本地权限代理任意执行 PowerShell、CMD、Shell 或 AppleScript。
- 不绕过验证码、平台风控、系统权限和登录保护。
- 不静默发布内容、上传文件、发送消息或替用户做付款决定。
- 不直接永久删除用户文件；删除默认进入系统废纸篓/回收站并尽量支持撤销。
